# 《重生之贵妇》5 分钟 Demo · 三方案对比

> 基于 Act I（雨夜纳妾 → 梦醒十年 → 三门送别 → 归来入帐预告）
> 三种呈现形式：**全静态 AVG** / **静态为主 + 动态点缀** / **动态为主 FMV 互动短剧**

---

## 0. 三方案总览对比

| 维度 | 方案 A：全静态 AVG | 方案 B：静态 + 动态点缀 | 方案 C：动态为主 FMV |
|---|---|---|---|
| 品类归属 | 传统视觉小说 | Cinematic Visual Novel | 第三人称互动短剧 |
| 视频占比 | 0% | 15–20% | 75–85% |
| 静态立绘占比 | 90% | 70% | 10–15% |
| 静态 CG 占比 | 10% | 10% | 0–5% |
| 参考作品 | 橙光/《魔法使之夜》 | 《隐形守护者》进化版 | 红果 AI 短剧 + 选项 |
| 目标用户 | 传统 gal / 橙光玩家 | AVG 老玩家 + 观感升级需求 | 短视频用户 + 剧集用户 |
| 5 分钟 Demo 成本 | **500–900 元** | **1000–1500 元** | **1500–2500 元** |
| 一人制作周期 | 1–2 周 | 2–3 周 | 3–4 周 |
| 正式版扩展成本 | 低 | 中 | 高 |
| AIGC 技术难度 | ★★☆☆☆ | ★★★☆☆ | ★★★★☆ |
| 观感"新鲜度" | ★★☆☆☆ | ★★★★☆ | ★★★★★ |
| Steam 传播上限 | 中 | 高 | 极高 |
| 主要风险 | 显"廉价"、被同类淹没 | 视频/静态风格不统一 | AI 视频一致性、成本超支 |

**一句话决策**：
- **预算 < 1000 元 / 想两周内做完** → 方案 A
- **想要"电影感"但不想全程做视频** → 方案 B（**综合推荐**）
- **想抢"AI 互动短剧"品类空白 / 追求最高观感** → 方案 C

# 方案 A：全静态 AVG（传统视觉小说）

## A1. 定位

- **形式**：文字对白 + 静态立绘（表情差分）+ 静态 CG + BGM + 配音
- **参考**：橙光文字游戏商业化版本、《寻沁记》早期版、《命运石之门》类
- **观感对标**：类似 2015–2020 年国产商业 AVG 水平
- **卖点**：剧情厚度、选项深度、抽卡式 CG 收藏

## A2. 内容结构（5 分钟切分）

| 时间 | 表现形式 | 内容 |

| 1:20–1:40 | **选项 1** | 雨夜回应（3 选项） |

| 4:30–4:40 | **选项 3** | 面对纪纤纤 |

## A6. 制作周期

## A7. 优缺点

**优点**
- 成本最低，风险最小
- AI 出图技术已经完全成熟，无翻车风险
- 玩家习惯这个形式，理解门槛为零
- 正式版扩展容易（每章 +30 张立绘 +5 张 CG）

**缺点**
- **观感"廉价"**——2026 年的 Steam 玩家已经被 AI 视频宠坏
- 静态画面缺乏电影感，卖不出高价（Steam 定价上限 30 元）
- 短视频引流困难（静态图剪不出爆款 30 秒）

## A8. 适合谁

- 想快速验证故事、看市场反馈的独立开发者
- 预算紧张，把资源留给正式版
- 目标用户是**橙光/传统 gal 死忠**，不追求电影感

# 方案 B：静态为主 + 动态点缀（Cinematic Visual Novel）

## B1. 定位

- **形式**：文字对白 + 立绘 + **关键节点插入 5–8 秒动态 CG（I2V 视频）**
- **参考**：《隐形守护者》图片版的进化形态、部分日式 AVG 的 OP+CG 动画
- **观感对标**：类似 2024 年最新一批"电影感 AVG"
- **卖点**：静态段落省成本，关键情绪点用动态视频拉升观感

## B2. 内容结构

| 时间 | 表现形式 | 内容 |
|---|---|---|
| 0:00–0:15 | **动态 CG-01（8s）** + 独白 | 雨夜白衣跪影动态镜头 |
| 0:15–0:50 | 对白 + 立绘 | 蜀王府内室对话 |
| 0:50–1:00 | **动态 CG-02（6s）** | 殷蕙"我不同意"面部特写 + 唇动 |
| 1:00–1:20 | **选项 1** | 雨夜回应 |
| 1:20–1:35 | **动态 CG-03（8s）** | 雨水墨黑吞噬 + 铜镜浮现（转场） |
| 1:35–2:20 | 对白 + 立绘 | 澄心堂重生对话 |
| 2:20–2:30 | **动态 CG-04（8s）** | 抱衡哥儿襁褓侧脸 |
| 2:30–2:50 | **选项 2** | 今生第一念 |
| 2:50–3:30 | 群像立绘 + 对白 | 侧殿群像（浓缩） |
| 3:30–3:40 | **动态 CG-05（6s）** | 燕王低机位入场 |
| 3:40–4:00 | **选项 3** | 面对纪纤纤 |
| 4:00–4:40 | 对白 + 立绘 | 魏曕安抚 |
| 4:40–5:00 | **动态 CG-06（10s）** | Cliffhanger 双人对视 |

**动态视频总时长 ≈ 46 秒 / 6 段**

## B3. 素材清单

- **立绘**：15 张（比方案 A 减少 7 张，因为 6 段视频替代了大量情绪转折立绘）
- **场景背景**：4 张（同 A）
- **动态 CG**：6 段 I2V 视频（**核心新增**）
- **动态首帧图**：6 张（用于 I2V 输入）

## B4. Prompt 库（新增动态视频部分）

**通用视频生成流程**：
```
Step 1: 即梦 4.0 生成"动态首帧图"（高质量单帧）
Step 2: 首帧图 + 动态描述 → 可灵 2.1 I2V → 5-8 秒视频
Step 3: 可灵内置对口型（说话镜头）或 Sonic 单独处理
Step 4: 剪映加转场/BGM/字幕
```

### 动态 CG Prompt 示例

**动态 CG-01：雨夜白衣跪影**

首帧图 prompt（即梦）：
```

```

**优点**
- 观感一步跨越到 2026 年水平（雨夜镜头 / 铜镜转场 / 对视 Cliffhanger 全动态）
- 静态部分省成本，动态部分只做 6 段（可控）
- 短视频引流有素材（6 段视频剪 30s 传播片段）
- **风险可控**——单段视频崩了不影响整体，只影响那 6–8 秒

**缺点**
- 静态和动态之间的画风必须严格统一（工笔水墨风），否则跳戏
- 需要同时掌握 3 个工具链（即梦 + 可灵 + Sonic），学习成本略高

## B8. 适合谁

- **多数独立开发者的最佳选择**
- 想要电影感但不想赌 AI 视频全流程
- 目标 Steam 定价 30–50 元区间
- 追求综合性价比

# 方案 B 极简版（500 元预算）

## B*1. 定位

- **形式**：方案 B 的极致做减法版本
- **核心思路**：**砍剧本、砍角色、砍场景**——把有限预算集中在 3–4 段关键动态视频上
- **观感目标**：依然保持"电影感 AVG"感觉，不退回全静态
- **预算硬约束**：≤ 500 元

## B*2. 核心修改策略

### 策略 1：砍剧本——从 3 场戏减到 2 场

**删除侧殿群像戏（SC-003 三门送别）**：
- 侧殿戏需要 4 个新角色（纪纤纤、徐清婉、燕王、温夫人）
- 群像视频段（燕王低机位入殿）AI 视频最难做，一致性会崩
- 5 分钟内塞 3 场戏，玩家来不及消化

**改为 2 场核心戏**：
```
Scene 1（0:00–2:00）雨夜纳妾（前世）
Scene 2（2:00–5:00）梦醒十年 + Cliffhanger
```

**过渡处理**：Cliffhanger 直接接在澄心堂之后，用一句独白交代王府线：

> 【画面切到 Cliffhanger】

- **传播效率极高**——25 段视频剪出 5 支短视频引流轻而易举
- **品类空白**——2026 年 Steam 上"第三人称 AI 互动短剧"这个位置没人占
- **Steam 定价可以拉到 50–80 元区间**

**缺点**
- **一致性风险高**——25 段视频里女主的脸必须严格保持一致，否则观感崩塌
- **可灵/Vidu 服务波动**——429 限流、错误码 414 等（你之前 pipeline.py 里踩过的坑）都会遇到
- **单段废稿率 30–50%**——每段视频可能要抽 2–3 次才有满意结果
- **剪辑工作量大**——25 段视频拼接 + 转场 + 音频对齐是硬工作

## C8. 适合谁

- 有 1 个月时间、2000 元预算的独立开发者
- 想要**观感/传播/商业价值三者最优解**
- 目标 Steam 定价 50–80 元 + 短视频引流
- 能接受 AI 视频抽卡的不确定性

## C9. 风险预案

### 一致性崩溃兜底

- 主角脸型漂了 → 用 **Runway Act-One** 或 **Hedra Character-3** 换脸重生成
- 关键情绪镜头（如"我不同意"）**抽卡 5 次以上**，选最好的
- 建立 **A/B 备用素材**，每个关键镜头存两版

### 服务稳定性兜底

- 可灵限流 → 用 **Vidu Q1** 或 **Seedance 1.0 Pro** 备份
- 即梦服务波动 → 用 **Flux.1 Kontext** 本地部署备份
- Sonic 效果不好 → **LivePortrait**（开源）本地兜底

### 时间/预算超支兜底

- 如果 3 周做不完，**砍到 15 段视频**，其余用静态过渡（降级为方案 B）
- 如果预算超到 3000 元，**减少抽卡次数**（每段抽 2 次而非 3 次）

# 决策建议

## 按目标场景推荐

| 你的目标 | 推荐方案 |
|---|---|
| 只想验证故事能不能打动人，两周做完 | **方案 A** |
| 想上 Steam 卖 30–50 元，观感够用 | **方案 B**（综合推荐） |
| 想抢品类空白，做 Steam / 短视频双平台爆款 | **方案 C** |
| 完全没有 AI 工具使用经验 | **方案 A** 起步 |
| 已经会用即梦、可灵、剪映 | **方案 B 或 C** |
| 一个人做 | **方案 A 或 B** |
| 有 2–3 人小团队 | **方案 C** |

## 综合最优：方案 B

我个人推荐 **方案 B（静态为主 + 动态点缀）** 作为多数独立开发者的最优解。

理由：
1. **成本可控**（1000–1300 元 vs 方案 C 的 2000+）
2. **风险可控**（6 段视频崩了不致命，方案 C 25 段崩一段就跳戏）
3. **观感足够**（关键情绪点全动态，玩家不会觉得"廉价"）
4. **周期合理**（2–3 周 vs 方案 C 的 3–4 周）
5. **可平滑升级到方案 C**（先做 B，如果反响好，正式版加更多动态 CG 就是 C）

## 综合路线建议

如果你想**分阶段验证**，建议这样走：

```
Phase 1（1 周）：先用方案 A 出一个 3 分钟原型
                验证故事、选项、剧情流是否成立
                成本 300 元

Phase 2（2 周）：升级到方案 B，做完整 5 分钟 Demo
                加入 6 段动态 CG
                总成本 1200 元

Phase 3（可选，投资/发行确认后）：
                升级到方案 C 全动态版本
                作为正式版的"章节 1"
                成本 2500 元
```

**这样每一步都有交付物，风险最小，回旋余地最大**。
